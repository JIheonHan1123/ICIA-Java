1. 자바의 특징
    - 어떤 시스템에도 실행이 가능하다 -> 호환성 -> 백엔드에서 널리 사용됨 
    - 자동 메모리 회수
    
    
2. 자바의 종류
    - JDK: 자바로 개발하기 위해
    - JRE: 자바로 작성한 코드를 실행
    				   JVM(Java Virtual Machine): 자바가상머신
    				   
    				   
3. 유료와 무료
    - 자바의 소유자는 오라클
    - 자바의 표준: Open JDK 공개
    - Open JDK의 오라클 구현 버전이 유료화
    
    
4. 버전
    - Java 9월부터 6개월마다 새로운버전이 나오고 기존 버전 폐기
    - 중간중간에 수명 3년짜리 버전들 운다.(Long Term Service)   
    
    
5. 패키지
    - 클래스 이름은 겹칠 수 있다. -> 클래스를 저장하는 폴더(패키지)를 만들자
    - 그런데 자바는 필요하면 만드는 것이 => 남이 만든 걸 가져다 쓴다
    - 그러니까 패키지 이름이 절대 겹치면 안된다.
    - 그래서 자신이 소속된 기관의 주소를 거꾸로 뒤집는다. (최소 .2개 이상)
    - ex) ai.samsung.co.kr -> kr.co.samsung.ai
